# OpenClaw Source Proof Checklist

Status: ACTIVE

## Goal

External monitored source signals must become traceable CRM and Telegram proof, not loose notifications.

## Required proof

- raw source reference
- source/channel name
- extracted service need
- area or ZIP if available
- confidence or rule hit
- dedupe key
- CRM write result or skip reason
- Telegram proof or failure
- follow-up status

## Implementation notes

No OpenClaw package or component is currently present in this repository. This checklist applies to the integration contract when the actual monitoring component/repo is connected.

## Stop rule

Do not claim OpenClaw lead monitoring works unless raw source, CRM event, and Telegram proof are all present.
