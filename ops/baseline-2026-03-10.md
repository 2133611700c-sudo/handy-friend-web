## Migration 023 Summary

Applied: 2026-03-10

1. **calc_real_response_min()** — removed `contacted_at` fallback; events-only
2. **Cleanup** — cleared false zeros from `response_time_min` (NOT EXISTS pattern)
3. **v_response_sla** — DROP+CREATE with `sla_source` diagnostic column
4. **refresh_daily_stats()** — dual-source `leads_contacted` (contacted_at + lead_events)
5. **dashboard_stats()** — removed redundant `> 0` filter + `stage NOT IN ('new')`

### Bugs Found During Apply
- PostgreSQL `NOT IN (subquery)` returns NULL when any subquery row has NULL `lead_id` → used `NOT EXISTS` instead
- `CREATE OR REPLACE VIEW` cannot rename columns → must `DROP VIEW` + `CREATE VIEW`
