# OpenClaw Skills

This directory defines execution playbooks for OpenClaw as ChatGPT's external
execution layer.

## Files

- `task-intake.md` — task loading, validation, and safety gate rules.
- `evidence-standard.md` — required evidence payload and report shape.
- `browser-audit.md` — browser audit procedure and checks.
- `repo-audit.md` — read-only repo audit workflow.
- `repo-patch.md` — scoped patch workflow and verification gates.
- `supabase-ops.md` — safe DB/report checks and redaction rules.
- `chatgpt-handoff.md` — structured handoff back to ChatGPT.
- `failure-taxonomy.md` — canonical failure classes.
- `evidence-retention.md` — commit vs artifact retention policy.
